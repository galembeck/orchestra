# CLAUDE.md — apps/admin

Internal backoffice dashboard for the Orchestra platform. Built with Vite + React 19 + TanStack Router (file-based) + TanStack Query.

## Commands

```bash
pnpm dev       # dev server on port 5173
pnpm build     # tsc -b && vite build
pnpm lint      # eslint
pnpm preview   # preview production build
```

Run Biome from this directory before considering any file done:

```bash
pnpm dlx @biomejs/biome check --write <file1> <file2> ...
```

## Path alias

`@/` resolves to `src/`. Example: `import Foo from "@/components/atoms/foo/foo"`.

## Folder structure

```
src/
  app.tsx                  # Router + QueryClient + AuthProvider bootstrap
  route-tree-gen.ts        # Auto-generated by TanStack Router — never edit manually

  pages/                   # File-based routes (routesDirectory for TanStack Router)
    __root.tsx             # Root route: ThemeProvider, HeadContent, Toaster
    _auth/                 # Pathless layout group — unauthenticated routes
      _sign-in/            # Nested pathless segment; resolves to "/"
        layout.tsx         # Sign-in shell layout
        index.tsx          # Sign-in page component
        ~components/       # Route-private components (prefixed "~", not picked up as routes)
    _app/                  # Pathless layout group — authenticated routes
      panel/               # Resolves to "/panel"
        layout.tsx         # Auth guard (redirects to "/" if unauthenticated) + sidebar shell
        index.tsx          # /panel index
        overview/          # /panel/overview
        validation/        # /panel/validation
        ~components/       # Route-private sidebar components

  components/              # App-local shared components (not in @repo/ui)
    atoms/                 # Small, reusable building blocks scoped to this app

  constants/               # Static data and configuration objects
    _app/
      panel/               # Sidebar nav items and search navigation data
```

## Routing conventions

TanStack Router generates `route-tree-gen.ts` automatically — never edit it.

- **Pathless segments** (`_auth`, `_app`, `_sign-in`) group routes without adding a URL segment.
- **Layout files** (`layout.tsx`) define `createFileRoute` with the pathless segment id (e.g. `"/_app/panel"`).
- **`routeToken: "layout"`** in `vite.config.ts` means the layout file inside a directory is the layout route, not an index.
- **`~components/`** directories are co-located private components for a route. The `~` prefix prevents TanStack Router from treating them as route files.
- **`autoCodeSplitting: true`** — every route's component is automatically code-split; import errors cause "Failed to fetch dynamically imported module" at runtime.

## Authentication

The panel layout (`_app/panel/layout.tsx`) guards all `/panel/*` routes via `beforeLoad`. It calls `adminAuthQueryOptions` (from `@repo/core/services/admin/admin-auth.service`) and redirects to `"/"` on a 401. No additional auth wrappers are needed inside panel routes.

## API

All HTTP calls go through `adminApi` from `@repo/core/api/connection/api-admin`. Do **not** create a local `lib/api.ts` — that file was intentionally removed in favour of the shared core package.

```ts
import { adminApi, type ApiException } from "@repo/core/api/connection/api-admin";
```

## Navigation / sidebar data

`src/constants/_app/panel/` holds the sidebar item list (`panel-sidebar.ts`) and the command-palette search items (`panel-navigation.ts`). When adding a new panel route, register it in **both** files.

## Form validation

Inline Zod schemas live in the same file as the form component (co-located). Do not create a separate `schemas/` folder inside this app.
